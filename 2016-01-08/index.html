Genesis 2.2.6 (and WordPress 4.4.1)

Genesis 2.2.6 has been released.

Usually for these more ‘minor’ maintenance releases, StudioPress doesn’t go into a lot of details about the changes, but this time they did because there’s a bit more going on.

While youâ€™re in there updating to WordPress 4.4.1, go ahead and update to #GenesisWP 2.2.6 too https://t.co/p7mL6DcyiC

— Nathan Rice (@nathanrice) January 6, 2016

If you’re more of a Genesis user than a developer then there’s nothing to worry about or concern yourself with.

I don’t want to repeat everything here butÂ if you just want the quick details, they are:

  • Updating Schema.org integration in Genesis to recognise individual pieces of contentÂ as ‘CreativeWork’ and not ‘blogPosting’ by default.Â For the latter, Google requires an associated Image and Publisher object, which is not always present and in some cases can cause content to be flagged as ‘spam’ by Google. (You still have the ability to override and markup any type of content (e.g. a custom post type as a ‘blogPosting’ or anything else you prefer).
  • Compatibility with how WordPress 4.4 handles storing meta data for terms.
  • Switching to using a local html5shiv file (which makes HTML5 elements play nicely on older web browsers) instead of pointing to the one hosted by Google Code, which is shutting down.

If you’re interested in the whole story though I encourage you to check out Nathan Rice’s article on Genesis 2.2.6.
