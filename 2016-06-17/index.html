This Week in Genesis (17 Jun 2016)

17 June 2016 by Nick Davis


Photo by Unsplash

This Week in Genesis, 2.3 lands, new Genesis contributors and a small but handy new plugin from Bill Erickson.

Genesis 2.3 (and new contributors!)

Yup, it’s the latest and greatest release of the Genesis Framework, version 2.3.

Genesis 2.3 now available. Update now with one-click! https://t.co/SSARqViVzc #genesiswp pic.twitter.com/WqEreEevuo

— StudioPress (@studiopress) June 15, 2016

Named Nick in honour of my greatness (OK, I made that up), Genesis 2.3 focuses on a “backlog of bug reports and enhancement suggestions” rather than big new features but, nevertheless, it’s a very welcome release.

And it seems to be going down well…

Well, first @WooCommerce update, then #GenesisWP update. Site still running smooth as silk even with 50 active plugins 🙂

— BobWP (@bobWP) June 16, 2016

@studiopress thanks for the update to #genesiswp, now at 2.3.0. Appreciate your continuing focus on this great WordPress theme!

— Neville Hobson (@jangles) June 16, 2016

Looks like I picked a good time to run client updates. Shiny new #Genesis 2.3 just became available #genesiswp

— Digital Davidson (@digidavidson) June 15, 2016

The only slight confusion seems to be the removal of the CSS column classes from the framework itself (a good thing). From this thread in the Facebook group, it looks like a few people were pulling these classes in ‘incorrectly’ from the Genesis Framework directly (instead of putting them in a child theme).

If you or a client is affected by that though, there’s a simple to handle this. Simply add the CSS classes at the following link (thanks to Anita for sharing the link in the Facebook group) to your child theme.
